# Название проекта	
Распознавание орфографических ошибок в рукописном тексте

# Проблема 
Учителя и школьники тратят много времени на ручную проверку орфографии. Готовые OCR-системы не распознают русский почерк с достаточной точностью, а использование мощных облачных ИИ (Gemini, GPT-4V) в школах ограничено из-за высокой стоимости, низкой скорости и требований к приватности данных. 

# Целевая аудитория 
Учителя, проверяющие рукописные работы учеников (диктанты, изложения, сочинения)
Школьники, самостоятельно проверяющие свои работы перед сдачей.

# Пользовательский сценарий 
Шаг 1: загрузка изображения рукописного текста в чат-бот в Telegram
Шаг 2: распознавание текста и орфографических ошибок
Шаг 3: вывод пользователю исходного текста и исправленных ошибок


# Какую задачу решает ИИ	
Распознавание рукописного текста (fine-tuned TrOCR), исправление орфографии выполняется локальной библиотекой JamSpell, Gemini используется только для разовой разметки датасета.

# Почему без ИИ решение хуже	
Обычные технологии оптического распознавания символов (OCR) дают низкую точность на русском рукописном тексте (ошибка >30–40%). Внедрение ИИ поможет с увеличением точности и понимания контекста.

# Тип AI-адаптации (один из 5)	
Fine-tuning (дообучение TrOCR на синтетических данных от Gemini)

# Baseline-решение
Tesseract OCR без дообучения + словарная проверка орфографии (Hunspell)

# Целевые метрики 
Скорость (latency): ≤3 секунды на одно изображение.   
Точность распознавания символов (CER): ≤8% (лучше чем Tesseract с 30-40%).   
Точность распознавания слов (WER) ≤20%  
Точность исправления орфографии (F1): ≥85%.   

# MVP на 4-6 недель 
1. Сбор 500 – 1000 изображений русского рукописного текста. 
2. Разметка через Gemini API (получение транскриптов)
3. Fine-tune TrOCR на этих парах (изображение – транскрипт) 
4. Простой Telegram-бот для загрузки фото. 
5. Интеграция fine-tuned модели TrOCR + JamSpell (локальная орфография)
6. Вывод исходного и исправленного текста.

# Риски 
Технические: Gemini API может быть медленным (>3 сек), нет денег на GPU для fine-tune, Gemini может ошибаться в разметке, JamSpell может не справляться со сложными контекстными ошибками. 
Данные: качество разметки зависит от Gemini, трудно найти размеченный датасет русского рукописного текста, почерк сильно варьируется. 
Безопасность: передача фото учеников через API — вопросы к согласию и хранению.

# Инструкция по запуску baseline
Скрипт [baseline_tesseract](src/baseline_tesseract.py):
- Загружает тестовые кейсы из [queries_answers](data/test_cases/queries_answers.json)
- Распознаёт текст через Tesseract OCR на русском языке
- Сравнивает с эталоном (вручную расшифрованные тексты)
- Вычисляет **CER** (символьную ошибку) и **WER** (словесную ошибку) для каждого фото
- Записывает результаты и распознанный текст в [baseline_results](data/test_cases/baseline_results.json)

**Результаты Baseline (Tesseract) на 20 тестовых изображениях:**
| Метрика | Среднее значение |
|---------|------------------|
| CER (символьная ошибка) | 99.19% |
| WER (словесная ошибка) | 104.57% |
